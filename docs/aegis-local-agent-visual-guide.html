01. Field brief

Aegis turns a local model stack into a credential-defense checkpoint.

This page is both a user guide and a demo aid. It explains what the project protects, how the sentinel is wired, which claims are certified today, and which checks prove that prevention happened before model completion.

CIFT

Model-specific hidden-state detection for exfiltration intent. The current certified reference path is Qwen/Qwen3-4B on MPS with a promoted runtime artifact.

certified
DP-HONEY

Substitutes synthetic canary credentials into declared credential slots and records them in the canary registry for later leakage detection.

active
Gateway

OpenAI-compatible proxy that normalizes turns, enforces CIFT and egress policy, forwards safe requests, and preserves audit context.

wired
NIMBUS

Deterministic canary and session-leakage critic for demo evidence, with the learned paper-faithful critic still treated as a future hardening path.

bounded
Observability

Readiness probes, capabilities output, smoke reports, audit JSONL, and a local console for operator-facing debugging during a demo.

visible

06. Safe modes

Keep the release claim sharp: certified where proven, bounded everywhere else.

CIFT support is model-specific. A new local model needs its own calibration, sealed/live evidence, and promoted runtime artifact before CIFT enforcement can be claimed for that model.

Certified CIFT

Reference path: Qwen3-4B on MPS

  • Sidecar attests the certified model id and immutable revision.
  • Tokenizer, chat template, layer/window, readout count, device, and dtype match the artifact.
  • CIFT can block exfiltration intent before provider generation.
Black-box mode

For models that have not passed certification

  • DP-HONEY, provider egress, canaries, deterministic NIMBUS, policy, and audit remain active.
  • CIFT support tier must be shown as unsupported.
  • Protected workflows still fail closed when required credential-slot metadata is missing.
Capstone demo

What this page can honestly show

  • Local OpenAI-compatible gateway in front of a self-hosted model.
  • Qwen3-4B CIFT pre-generation block evidence with MPS device attestation.
  • Honeytoken substitution, provider egress guard, canary detection, policy, and audit output.
Not claimed

What remains outside the current release claim

  • Universal CIFT support across arbitrary models without their own certification run.
  • Commercial production readiness or external live deployment evidence.
  • Fully learned paper-faithful NIMBUS as the default runtime critic.

This guide is for local-agent integration. It does not make a universal model-support claim: each local model needs its own CIFT calibration, sealed/live evaluation, and promoted runtime artifact before CIFT enforcement can be claimed.
