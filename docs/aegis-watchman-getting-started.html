00. Which guide do you need?

Use one page, but keep two reading modes.

A first-time user needs a short path to protection. An operator needs to know what each control, warning, smoke result, and audit record means.

Quick start

01. Before you start

The launcher expects a local model provider and the Aegis repo.

The reference setup is a local development and demo profile. Keep Ollama, Aegis, and Hermes on the same machine unless you are deliberately testing a different network boundary.

Need Why it matters
macOS 14 Sonoma or later The current Ollama macOS download page lists macOS 14 Sonoma or later.
GPU runtime The certified Qwen3-4B reference path uses MPS. Other devices require their own certification evidence.
Ollama installed Provides the local OpenAI-compatible provider endpoint that Aegis forwards allowed requests to.
Hermes Agent installed Acts as the user-facing agent app for this guide. Other OpenAI-compatible agents can be configured later.
Aegis repo checked out Runs the launcher, gateway, console, CIFT sidecar scripts, and smoke checks.
Qwen3-4B CIFT artifacts The launcher uses the promoted runtime binding for Qwen/Qwen3-4B. Do not reuse those artifacts for a different model.

02. Quick start

Use the launcher for the normal path.

The launcher turns the old command sequence into one guided flow. Use the manual commands only when you are debugging the launcher or explaining the underlying services.

1

Install or open Ollama

Use the official macOS app, or use the install script if you prefer a terminal setup. Opening the app normally starts the local server.

Ollama install and server check 
curl -fsSL https://ollama.com/install.sh | sh
open -a Ollama
curl -s http://127.0.0.1:11434/api/tags
2

Make sure Qwen is available

If qwen3:4b is already installed, use it. If multiple models are installed, choose the right served model in the launcher.

Pull only when needed 
ollama list
ollama pull qwen3:4b
3

Start the Aegis launcher

Run this from the Aegis repository root. The desktop app checks or starts Ollama, pulls the selected model when needed, starts the shared launcher backend, and keeps the default Qwen3-4B profile visible.

Open desktop launcher 
scripts/run-aegis-desktop-launcher.sh
Build local app 
cd desktop/aegis-launcher
npm run pack:mac
open "dist/mac-arm64/Aegis Watchman.app"
Browser fallback 
uv run aegis-launcher \
  --host 127.0.0.1 \
  --port 8790
4

Click Start Protection

The desktop timeline checks the provider, model, machine, GPU, CIFT sidecar, gateway, and console. Already-running or already-installed items are skipped.

5

Click Run Verification

Run verification after CIFT and the gateway are running. You want one benign allow, one exfiltration block before provider completion, and audit evidence that names the detector.

03. Launcher user guide

The desktop app is the normal path. The browser launcher is the fallback.

The launcher is the user-facing setup app. It should answer three questions: what is configured, what is running, and what evidence exists.

Desktop

Preferred launcher

The macOS app owns the window, starts Ollama, pulls models, and calls the shared Python backend for Aegis services.

Normal

Start Protection

Runs the default path in order. The timeline reports skipped, running, passed, failed, and elapsed time for each step.

Normal

Run Verification

Runs smoke evidence after protection is running. This is the demo proof path, not a replacement for certification.

Fallback

Browser launcher

Use the browser launcher when you need the older manual controls or want to debug the shared backend directly.

Control What it means
Provider prerequisites Shows Ollama reachability, model selection, and machine warnings. The desktop app can start Ollama and pull the selected model.
Model picker Appears when more than one Ollama model is installed. Pick the served model Aegis should forward allowed requests to.
Start Protection Runs the stack. If protection is already running, the button becomes Stop Protection.
Run Verification Runs live smoke checks after the stack is running. Do not treat offline replay as production evidence.
Clear Statuses Clears old launcher statuses after services are stopped. Use it before starting a clean run.
CIFT Certificate Displays locked model-specific binding information: model id, revision, tokenizer/template hashes, layer/window, device, and dtype.

04. Point Hermes Agent at Aegis

Test Hermes with Ollama if you need a baseline, then switch it to Aegis.

Hermes uses ordinary OpenAI-compatible provider settings. Direct Ollama is useful for confirming the model works; protected operation means Hermes calls Aegis.

Optional baseline: Hermes directly to Ollama

Use this only to confirm that Hermes can talk to the model you pulled with Ollama. There is no Aegis protection on this path.

Provider type Custom OpenAI-compatible
Base URL http://127.0.0.1:11434/v1
API key ollama
Model qwen3:4b
Copy direct Ollama settings 
Provider: Custom OpenAI-compatible
Base URL: http://127.0.0.1:11434/v1
API key: ollama
Model: qwen3:4b

Direct Ollama is not protected.

Use the direct settings only as a model baseline. Switch Hermes to the Aegis base URL before testing secrets, canaries, CIFT, policy, or audit.

Protected path: Hermes to Aegis

After the launcher reports protection running, replace the direct Ollama base URL with the Aegis gateway URL.

Provider type Custom OpenAI-compatible
Base URL http://127.0.0.1:8000/v1
API key aegis-local-dev-key
Model qwen3:4b
Copy into Hermes settings 
Provider: Custom OpenAI-compatible
Base URL: http://127.0.0.1:8000/v1
API key: aegis-local-dev-key
Model: qwen3:4b

Do not point Hermes directly at Ollama while testing Aegis.

If Hermes uses http://127.0.0.1:11434/v1, it bypasses DP-HONEY, CIFT, egress guard, canary detection, NIMBUS, policy, and audit.

05. Evidence

A good run proves both allow and block.

The goal is not a green UI. The goal is a security layer that allows benign traffic, blocks exfiltration intent before provider completion, and records why.

Check Evidence you want
Benign request provider_status="completed". This proves Aegis is a live proxy, not a blanket blocker.
Exfiltration-intent request final_action="block" and provider_status="skipped". This proves pre-generation prevention.
CIFT support Runtime-enforceable sidecar evidence with matching model id, revision, hashes, layer/window, device, and dtype.
Audit trail A redacted JSONL record that names detector results, policy action, trace id, and why the provider did or did not run.

06. Manual fallback

Use commands when you need to isolate one layer.

These commands are not the preferred user path. They are here so an operator can prove whether a problem is in Ollama, CIFT, the gateway, or the agent settings.

Install Aegis dependencies 
cd <path-to-aegis-repo>
uv sync --extra dev
uv run aegis-launcher --help
uv run aegis-proxy --help
uv run aegis-console --help
Direct request through Aegis 
curl -s http://127.0.0.1:8000/v1/chat/completions \
  -H "Authorization: Bearer aegis-local-dev-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3:4b",
    "messages": [{"role": "user", "content": "In one sentence, explain what Aegis does."}],
    "stream": false
  }'

Do not silently downgrade CIFT.

If hidden-state extraction, selected-choice metadata, attestation, or certificate binding fails, do not claim Qwen3-4B CIFT enforcement. Fix the binding or present CIFT as unavailable.

07. Troubleshooting

Most setup failures are one wrong endpoint or one degraded binding.

Fix the first failing layer before moving on. The intended route is provider first, CIFT sidecar second, gateway third, agent last.

Ollama does not answer on 11434

Open the Ollama app, then rerun the launcher check or curl -s http://127.0.0.1:11434/api/tags.

qwen3:4b is missing

If another installed model is intentional, choose it in the launcher provider card. Otherwise run ollama pull qwen3:4b.

Machine checks show warnings

Click Show in the Machine ready card. Warnings do not always block startup, but they should be understood before claiming a clean demo.

CIFT is unsupported or degraded

Confirm the sidecar is running, the GPU check passed, and the strict deployment env matches the certified artifact.

Hermes works but Aegis sees no traffic

Hermes is probably pointed directly at Ollama. Change the base URL to http://127.0.0.1:8000/v1.

Hermes says Aegis advertised no models

Restart the Aegis gateway, then check http://127.0.0.1:8000/v1/models. It should list the configured model, usually qwen3:4b.

Benign traffic is blocked

Check the audit record first. If the block is from CIFT binding failure, fix the sidecar rather than loosening policy.

08. Later setup pages

Once the reference path works, other stacks are substitutions.

The next pages should keep the same contract: agent base URL points at Aegis, provider credentials live in Aegis, CIFT support is claimed only after certification.

Provider swap

LM Studio, llama.cpp, or vLLM

Replace AEGIS_OPENAI_BASE_URL and AEGIS_OPENAI_MODEL, then rerun readiness and smoke checks.

Agent swap

Open WebUI or another local app

Use the same OpenAI-compatible fields: base URL, API key, and model. The agent should never own the detector configuration.

Model certification

Another hidden-state-capable model

Run the CIFT calibration and certification workflow. Do not reuse Qwen3-4B activation artifacts for a different model.

Black-box mode

No hidden states available

Keep DP-HONEY, provider egress guard, canary detectors, deterministic NIMBUS, policy, and audit. Mark CIFT unsupported.

Official references used by this guide: Ollama download, Ollama qwen3:4b, and Ollama OpenAI compatibility. Aegis CIFT support remains model-specific: Qwen3-4B is the certified reference path here, while other models need their own calibration, sealed holdout, live runtime, and gateway smoke evidence.
